Detailed Description

Loads and provides mappings of file-type categories to file extensions.

The module attempts to read data/filetypes.json for a user-provided mapping of category â†’ extension list. If the JSON file is missing or invalid the implementation falls back to a built-in mapping returned by fallbackFileTypes().

The getFileTypes() function provides a cached, read-once accessor that loads the mappings on first call.

std::map< std::string, std::vector< std::string > > loadFileTypes ( )

Loads file type mappings from data/filetypes.json.

Load file type mappings from disk (or fallback).

Attempts to open and parse the JSON file. The expected format is a top-level object where keys are category names and values are arrays of extension strings (including the leading dot), e.g.: { "Images": [".jpg",".png"], "Documents": [".pdf",".docx"] }

If the file cannot be opened or parsed, the function returns the fallbackFileTypes() mapping.

Returns
std::map<std::string, std::vector<std::string>> Loaded mapping.
